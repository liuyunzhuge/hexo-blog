## 奖章获得方式总结：
```
1、专门针对新人的奖章：完成一个特定的新人任务即可获得

2、触发特定的事件即可获得
    首次分享、首次消费、单次跑步平均配速低于xxx

3、指定时间段，完成某个目标，并累计xx天可获得
    夜幕游侠、午时行者、夜游神、夜猫子、早起鸟

4、最长连续练习周、最长连续练习天
    如连续n周完成某个任务，则可点亮最长连续练习周的奖章，往后继续连续每周完成相同任务
    则最长连续练习周这个奖章上的数字会增加

5、周目标、月目标、年目标
    完成周、月、年特定的目标，则可点亮该周、月、年的奖章

6、参与线上比赛、活动、完成挑战
    活动与挑战的概念有点模糊不清

7、节日、节气、纪念章

8、某个维度的累计
    如次数、小时、km、天数等

9、连续达成某个目标多少天、多少周
    升级型：
        连续3天 连续7天 连续15天

10、游乐园，大小奖章结合模式
    点亮所有小的奖章，才能点亮大的
```

## 奖章形式分类：
```
1、 单次永久型
    获得之后就点亮，之后再无变化
2、 升级型
    升级型获得方式相似，在展现的时候有两种形式：
    a. 像全是瑜现在全部展现，咕咚、悦跑圈都是这样的
    b. 统一把一个类型的升级型奖章展示为1个，默认显示用户点亮的最高级别
        得到在这种型展现的效果是最清晰的，keep也是这么做的

3、 连续型
    两种做法：
    a. 做成最长连续xx型
        如最长连续练习天，最长连续练习周，但是这种奖章在点亮后，还需要继续更改状态
    b. 做成升级型
        连续3天 连续5天 连续10天这种

4、 可重复获得型
    带有“每”字含义的奖章设计，通常都是可重复获得型，如
        每周练习3天、每节课完成5连对
    可重复获得型，一定有周期的限定，如每周、每天、每月、每次
    周期越短，则触发频率越高，这个地方需要衡量产品的人群对于频繁弹出奖章的感受

    重复型的奖章如何设计，突出获得次数，清晰的展示获得记录（时间轴）
```

## 主要的奖章灵感：
```
1、核心数据的积累、持续、连续程度
2、特殊事件的触发
3、社区活跃
4、线上活动、比赛、挑战
5、与城市地理相关
6、与节气、节日等传统文化相关
7、与星座相关
8、与十二章纹相关
9、有纪念意义的事件
10、假期行为
11、彩蛋
12、大小奖章组合
```

## 奖章的数据管理：
```
二级分类，如：
    岁月如歌：
        按节气、节日：
            庚子年节气 己亥年节气
            2020年节日 2019年节日
    精彩纷呈
        按年份：
            2020线上马 2019年线上马
            2020年活动 2019年活动
            2020年挑战 2019年挑战

有时效性：设置奖章可被点亮的时间范围，过期则不展示
```

## 亮点功能：
```
时光轴
新奖章轮播
奖章墙
奖励机制
实物奖章
获得人数的展示、排行榜
奖章在获得之前有提供入口进行引导，如
    keep的去获得，会跳转到相应的课程页面
    咕咚的去了解，会跳转到对应的资讯页面

    也就是说有对于不同奖章定义各自在点亮前后的交互行为
```
